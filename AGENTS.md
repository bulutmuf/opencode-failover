# opencode-failover - Agent Guidelines

## Release Management
When creating GitHub Releases for this repository, you must adhere strictly to the following conventions:
- **No Emojis:** Do not include any emojis in the release notes or the release title to prevent character encoding issues on certain platforms (like PowerShell).
- **Strict Title Format:** The release title must ONLY be the exact version number (e.g., `v2.0.7`). Do NOT append any explanations, colons, or feature summaries to the title.
- **Professional English:** Write release notes in professional English, structured with bullet points.
